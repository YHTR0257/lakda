# Feature Design: FT01 - 質問入力 (Question Input)

## 1. 機能概要 (Overview)

本機能は、ユーザーが専用の質問入力画面（`SC07`）を通じてシステムに質問を入力し、その質問がBackendで正常に受信されるまでを対象とします。

- **Feature ID**: FT01
- **Feature Name**: 質問入力 (Question Input)
- **Goal**: ユーザーが入力した質問をBackendが受け取れることを確認する。

## 2. 担当コンポーネント (Components)

- **Frontend**: ユーザーインターフェースを提供し、質問入力とAPIリクエストの送信を担当します。
- **Backend**: APIエンドポイントを提供し、Frontendからのリクエストを受信します。
- **State Management**: Zustand (`useAskStore`) を使用して、会話履歴やUIの状態を管理します。

## 3. UI/UX設計 (UI/UX Design)

### 3.1. 画面構成

本機能は、以下の2つの画面によって構成されます。

- **SC01: Main画面 (`/`)**:
  - システムのメインページ。
  - `SC02: ドキュメントアップロード画面` や `SC07: 質問入力画面` へのナビゲーションを提供します。
  - 「質問を開始する」などのボタンを配置し、`SC07`へ遷移させます。

- **SC07: 質問入力画面 (`/ask`)**:
  - Q&A対話専用のページ。
  - 画面下部に質問入力エリアと送信ボタンを配置します。
  - 画面の主要領域には、ユーザーの質問とシステムの回答の履歴を表示します（FT10で実装）。

### 3.2. ユーザーインタラクション

1.  ユーザーは`SC01: Main画面`の「質問を開始する」ボタンをクリックし、`SC07: 質問入力画面`へ遷移します。
2.  ユーザーは画面下部のテキストエリアに質問したい内容を入力します。
3.  テキストが入力されると、送信ボタンが活性化します。
4.  ユーザーが送信ボタンをクリックすると、入力内容がBackendに送信され、会話履歴にユーザーの質問が追加されます。
5.  送信中は、UI上でローディング中であることが示されます。

## 4. データフロー (Data Flow)

1.  **[Frontend]**: ユーザーが送信ボタンをクリックすると、`useAsk`フックが入力エリアから質問文字列を取得します。
2.  **[Frontend]**: `useAskStore`を通じて、ユーザーの質問を会話履歴の状態に追加し、UIに即時反映させます。
3.  **[Frontend -> Backend]**: `apiClient`を介して、質問文字列をJSONペイロードに含め、BackendのAPIエンドポイントにHTTP POSTリクエストを送信します。
4.  **[Backend]**: APIエンドポイントでリクエストを受信し、質問文字列を抽出します。
5.  **[Backend]**: 後続の処理（FT08, FT09）を実行し、生成された回答をレスポンスとして返します。
6.  **[Backend -> Frontend]**: `200 OK`ステータスと共に、回答データを含むJSONを返します。
7.  **[Frontend]**: `useAsk`フックがレスポンスを受け取ります。
8.  **[Frontend]**: `useAskStore`を通じて、システムの回答を会話履歴の状態に追加し、UIを更新します。エラーが発生した場合は、エラー情報を状態に保存します。

## 5. API仕様 (API Specification)

- **エンドポイント**: `POST /api/v1/ask`
- **メソッド**: `POST`
- **ヘッダー**:
  - `Content-Type: application/json`

### 5.1. リクエストボディ

```json
{
  "question": "Pythonで発生したAttributeErrorの原因と解決方法は？"
}
```

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `question` | string | Yes | ユーザーが入力した質問の文字列。 |

### 5.2. レスポンス (同期的な回答)

#### 成功時 (200 OK)

FT09で生成された回答と引用元を返します。

```json
{
  "answer": "AttributeErrorは、オブジェクトに存在しない属性にアクセスしようとしたときに発生します。...",
  "sources": [
    { "name": "python_exceptions.md", "url": "/docs/python_exceptions.md" }
  ]
}
```

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `answer` | string | 生成された回答の本文。 |
| `sources` | array | 回答の根拠となった引用元情報のリスト。 |

#### 失敗時 (400 Bad Request / 500 Internal Server Error)

```json
{
  "detail": "Error message explaining the issue."
}
```

## 6. 成功基準 (Acceptance Criteria)

- [ ] `SC01: Main画面`から`SC07: 質問入力画面`へ遷移できる。
- [ ] ユーザーは`SC07`のテキストエリアに質問を入力できる。
- [ ] 送信ボタンを押すと、入力された質問が会話履歴に追加され、BackendにAPIリクエストが送信される。
- [ ] BackendはAPIリクエストから質問文字列を正しく抽出し、同期的に回答を生成して返す。
- [ ] Frontendは成功レスポンスを受け取り、システムの回答を会話履歴に追加して表示する。
- [ ] APIリクエストや回答生成に失敗した場合、UI上でエラーが適切にハンドリングされる。
